# Handoff Report — Word Document Template

Generate a `.docx` using `python-docx`. Save to `~/Desktop/plcman-<PROGRAM>-<YYYY-MM-DD>.docx`.

## Document Structure

### Title
`PLC Review — <Program Name> <Version>`

### Subtitle
`Generated by plcman on <YYYY-MM-DD>`

### Summary Section

A single paragraph stating:
- Total tickets processed
- Passed / Failed / In Progress / Needs Action counts
- Show stopper count (if any)

### Summary Table

| Column | Width | Content |
|---|---|---|
| Ticket | 1.2" | Jira key (e.g., FVSDK-123) |
| Task | 1.5" | Task type (e.g., vuln-scan) |
| Tier | 0.5" | 1, 2, 3, or SKIP |
| Status | 1.0" | PASS / FAIL / IN PROGRESS / NEEDS ACTION / DONE / NOT APPLICABLE |
| Show Stopper | 0.8" | Yes / No |
| Action | 2.5" | What was done or what's needed |

### Row Color Coding

Use `python-docx` cell shading via `oxml`:

```python
from docx.oxml.ns import qn
from docx.oxml import OxmlElement

def set_cell_color(cell, hex_color):
    shading = OxmlElement('w:shd')
    shading.set(qn('w:fill'), hex_color)
    shading.set(qn('w:val'), 'clear')
    cell._tc.get_or_add_tcPr().append(shading)
```

| Status | Color | Hex |
|---|---|---|
| PASS / DONE | Light green | `C6EFCE` |
| FAIL (show stopper) | Light red | `FFC7CE` |
| FAIL (non-blocker) | Light orange | `FFE0B2` |
| IN PROGRESS | Light blue | `D6E4F0` |
| NEEDS ACTION | Light yellow | `FFFFCC` |
| NOT APPLICABLE | Light gray | `E0E0E0` |
| SKIP (release-review) | White | `FFFFFF` |

### Sort Order

1. FAIL (show stoppers) first
2. FAIL (non-blockers)
3. IN PROGRESS
4. NEEDS ACTION
5. PASS
6. NOT APPLICABLE
7. DONE (pre-existing)
8. SKIP (release-review — always last)

### Show Stopper Section

If any tickets are show stoppers with FAIL status, add a section after the table:

**Heading**: "Show Stoppers Requiring Immediate Attention"

Bulleted list, one per show stopper:
- `<TICKET-KEY>: <task type> — <what failed and specific remediation>`

### Actions Taken Section

**Heading**: "Actions Taken by plcman"

Bulleted list of what plcman actively did:
- Documents generated (with Confluence links)
- TAVA generated (with link)
- Bugs created (with NVBug IDs / Jira keys, linked to PLC tickets)
- Legal process initiated
- Comments posted to tickets

### Next Steps Section

**Heading**: "Remaining Actions for Human Review"

Bulleted list grouped by assignee:
- `@assignee: <TICKET-KEY> — <what they need to do>`

Final line: `Release Review (TICKET-KEY) — user to initiate after all above items are resolved.`

### Document Formatting

- Font: Calibri 11pt body, 14pt title, 12pt headings
- Table: borders on all cells, header row bold with dark gray background (`404040`, white text)
- Page: Letter size, 0.75" margins
- Footer: "plcman — PLC Task Executor | Confidential"

### Attach to Parent Ticket

After saving the `.docx`, attach it to the parent PLC ticket via Jira REST API:

```bash
source ~/.claude/jill/.env
curl -s -u "$JIRA_USERNAME:$JIRA_API_TOKEN" \
  -X POST -H "X-Atlassian-Token: no-check" \
  -F "file=@$HOME/Desktop/plcman-<PROGRAM>-<DATE>.docx" \
  "$JIRA_BASE_URL/rest/api/2/issue/<PARENT-KEY>/attachments"
```

Verify HTTP 200 response. The Jira MCP server does not support attachments, so this must use curl directly.
